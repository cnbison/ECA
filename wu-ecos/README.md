# ECOS 认知架构图集

> 用 [ECOS](https://github.com/cnbison/ecos)(教育认知操作系统)的理论,图解"**认知架构到底是什么**",并用一个具体案例——**学生学习 Claude Skills**——把抽象理论跑通。

ECOS 里"认知架构"不是一张模块连线图,而是一对紧耦合的东西:

- **状态表征(WHAT)** = `BeliefState`,一套带置信度的认知状态坐标系;
- **测量栈(HOW)** = CTA 的 L0–L5 数学栈,把行为反推成这个坐标。

二者合起来就是 **CTA(Cognitive Twin Agent)**。

## 三条贯穿全集的核心观点

1. **认知状态是"信念分布",不是"点"** —— 每个值都带置信度和证据;`A→B` 是分布的移动,不是两点相减。
2. **状态 = `资源(5D) × 深度(Bloom)` 的网格** —— 5D(K/P/S/C/X)说"用哪种本事",Bloom(Remember→Create)说"到什么火候";完整状态是 5×6=30 格。
3. **LLM 只碰两端,数值估计纯靠数学栈** —— LLM 只做感知层(行为→结构化)和解释层(统计值→人话);K/P/S/C 的数值由 BKT/MIRT/CD-CAT 估出,LLM 不参与。这正是 `A→B` 可信、可证伪的前提。

## 案例约定:学习 Claude Skills

- **达标线**:理解本质(`Understand ≥ 0.85`)+ 能在正确场景判断并写出最小可用 skill(`Apply ≥ 0.75`),不要求 Create 满级。
- **核心阈值概念(TC_skill)**:"skill 是模型按 description 相关性*自主*加载的能力,不是宏、不是外部进程"。
- **要清除的五个误概念**:M1 斜杠命令 · M2 MCP server · M3 任意自动化/hook · M4 prompt 模板 · M5 总是被加载。
- **达标四闸**:① TC 跨越　② Bloom U≥0.85∧A≥0.75　③ 误概念清零　④ C 是"挣来的"(伪置信=false)。
- **最硬的判定证据**:两个"否决"场景(该用 hook / 该用 MCP 的情况能正确拒绝并说对理由)——无法靠背定义伪造。
